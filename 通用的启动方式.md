# 通用的启动方式

适用于这类“前端 + 后端 + 可选附加服务”的本地开发项目。

当前项目对应关系：

- 后端：`backend/`，启动命令是 `python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000`
- 前端：`frontend/`，启动命令是 `npm run dev`
- 靶场：`sandbox/`，按需启动

---

## 1. 通用原则

建议顺序：

1. 先启动后端
2. 再启动前端
3. 最后启动附加服务（如沙箱、数据库、消息队列）

这样做的原因：

- 前端通常依赖后端接口
- 启动时更容易定位问题
- 端口冲突更好排查

---

## 2. 前台启动方式

适合调试，能直接看到日志。

### 2.1 启动后端

```powershell
cd D:\github_project\校赛\backend
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

### 2.2 启动前端

新开一个终端窗口：

```powershell
cd D:\github_project\校赛\frontend
npm run dev
```

### 2.3 按需启动靶场

再开一个终端窗口：

```powershell
cd D:\github_project\校赛\sandbox
python -m uvicorn agent_app.main:app --host 0.0.0.0 --port 18080
```

---

## 3. 后台启动方式

适合日常开发，不阻塞当前终端。

### 3.1 后端

```powershell
Start-Process -FilePath "cmd" -ArgumentList "/c python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000" -WorkingDirectory "D:\github_project\校赛\backend"
```

### 3.2 前端

```powershell
Start-Process -FilePath "cmd" -ArgumentList "/c npm run dev" -WorkingDirectory "D:\github_project\校赛\frontend"
```

### 3.3 靶场

```powershell
Start-Process -FilePath "cmd" -ArgumentList "/c python -m uvicorn agent_app.main:app --host 0.0.0.0 --port 18080" -WorkingDirectory "D:\github_project\校赛\sandbox"
```

## 4. 一键启动模板

如果以后别的项目也是类似结构，可以按这个模板改目录和命令。

```powershell
# 先停旧进程（按需）
Get-Process -Name "python" -ErrorAction SilentlyContinue | Stop-Process -Force -ErrorAction SilentlyContinue
Get-Process -Name "node" -ErrorAction SilentlyContinue | Stop-Process -Force -ErrorAction SilentlyContinue

Start-Sleep -Seconds 2

# 启动后端
Start-Process -FilePath "cmd" -ArgumentList "/c python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000" -WorkingDirectory "D:\github_project\校赛\backend"

# 启动前端
Start-Process -FilePath "cmd" -ArgumentList "/c npm run dev" -WorkingDirectory "D:\github_project\校赛\frontend"

# 按需启动附加服务
# Start-Process -FilePath "cmd" -ArgumentList "/c python -m uvicorn agent_app.main:app --host 0.0.0.0 --port 18080" -WorkingDirectory "D:\github_project\校赛\sandbox"
```

## 5. 启动成功检查

### 后端检查

```powershell
Invoke-WebRequest -Uri "http://127.0.0.1:8000/" -UseBasicParsing
```

预期返回状态码 `200`。

### 前端检查

```powershell
Invoke-WebRequest -Uri "http://localhost:5173/" -UseBasicParsing
```

预期返回状态码 `200`。

### 靶场检查

```powershell
Invoke-WebRequest -Uri "http://127.0.0.1:18080/health" -UseBasicParsing
```

预期返回状态码 `200`。

## 7. 停止方式

### 停止当前项目常用进程

```powershell
Get-Process -Name "python" -ErrorAction SilentlyContinue | Stop-Process -Force -ErrorAction SilentlyContinue
Get-Process -Name "node" -ErrorAction SilentlyContinue | Stop-Process -Force -ErrorAction SilentlyContinue
```

如果你不想误杀别的 Python / Node 进程，就直接关掉对应启动窗口。

## 8. 常见问题

### 8.1 前端能打开，但接口报错

一般是后端没启动，或端口不对。

先检查：

```powershell
Invoke-WebRequest -Uri "http://127.0.0.1:8000/" -UseBasicParsing
```

### 8.2 前端启动失败

先安装依赖：

```powershell
cd D:\github_project\校赛\frontend
npm install
```

### 8.3 后端启动失败

先安装依赖：

```powershell
cd D:\github_project\校赛\backend
pip install -r requirements.txt
```

### 8.4 端口被占用

查端口：

```powershell
netstat -ano | findstr ":8000"
netstat -ano | findstr ":5173"
netstat -ano | findstr ":18080"
```

### 8.5 修改代码后没生效

- 后端确认是否用了 `--reload`
- 前端确认是否是 `npm run dev`
- 浏览器强刷一次：`Ctrl + F5`

## 9. 适合以后复用的思路

以后遇到任何类似项目，都可以直接套这个结构：

1. 明确每个服务目录
2. 明确每个服务启动命令
3. 明确每个服务端口
4. 提供前台启动和后台启动两套方式
5. 提供健康检查命令
6. 提供停止方式

如果只是换项目：

- 把目录改掉
- 把启动命令改掉
- 把端口改掉

其余流程都可以不变。
